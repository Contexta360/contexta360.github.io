Description
The Create Query menu is very similar to the Selections menu in the Classic Dashboard. One major difference is that it also includes a box with interactions matching the query and not matching the query.

All the query functionalities from the Classic Dashboard should be present in the new one. The counts and percentages should also be shown as before:
Total count: Show the total number of conversations matching the query. Percentage: The percentage of conversations matching this query
Load query filter dropdown
Similar to the current Selections menu, it is possible to load an existing query. This can be with either a filter dropdown or a separate dropdown list and search box.
Test Query button
The Test Query button refreshes the Positive Interactions and Negative Interactions box.
Save button
Saves a new or existing query
Delete button
This is not in the sketch but there should also be a delete button for the queries.
Test Positive interactions
This is a list of interactions, similar to the lister on the agent page that match the query. This allows the user to quickly test the query. He can validate that the interactions that match the query are indeed relevant.
The positive interactions should have a search box (not in the sketch). This helps to add keywords as an additional filter. Endpoint: /lister/documents

{"filters" : [
{"queryTags" : [{"id" : "", "name" : ""}]},
{"searchTerm" : "apple" }
] }
Detail pages
When the user clicks on an interaction to go to the detail page, it is important that he can see the query mention as a waveform event.
Test Negative interactions
The same as positive interactions but this box shows all conversation that don't match. This allows the user to look for false negatives: conversations that are not matched by the query but could still be very relevant. By using the search box (search box only for now), the user can search for specific words.
Note: The only difference with the query of the positive interactions is a "negative" field on the selection level. Endpoint: /lister/documents

Queries can be saved and applied to metrics.
