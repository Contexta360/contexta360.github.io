This metric detects segments where agents are experiencing problems. For instance, when the agent's system is running slow or he has to call the client back due to an issue outside his control, such as:

%E2%80%A2 My system
%E2%80%A2 My computer
%E2%80%A2 Please hold
%E2%80%A2 Running slow
%E2%80%A2 I/We can't do that
Contact centre agents can be rewarded and recognised based on different KPIs. The manual scorecard approach or sampled analysis of voice recordings does not always illustrate the true meaning and understanding of a call. If an agent is being 'scored' based on average call handling times and is taking longer than the allotted time for a call, he may well be negatively viewed as there is no context as to why he has taken so long.

Therefore, it is key to understand if it is a technical or process issue that is preventing his ability to deal with his call within the target time.

This topic is trained with the Topic Labeller. Segments that contain agent problems are detected.Therefore, you will see events when the agent problems are below target.
